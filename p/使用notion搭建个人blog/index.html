使用notion搭建blog

为什么使用notion

现在建博客的工具有很多很多。

如果想要一个静态博客，你可以选择hugo、jekyll、hexo搭配github page实现。

如果想要一个动态博客，又没有什么基础，那你可以选择WordPress，网上教程一大把。

不过我采用的是notion＋github，并托管到vercel。

使用notion基于以下几点考虑：

  1. notion支持全平台(Windows、网页、Mac、ios、安卓)。可以随时随地地码字
  2. 页面简洁美观，功能强大。特别是database功能，非常方便管理，同时支持多种views展示
  3. 支持Markdown语法

使用到的工具：

  • notion 账号
  • github 账号
  • vercel 账号
  • git
  • node.js
  • vscode

使用模版

nextjs-notion-starter-kit

  1. fork 或 clone 项目

    • 使用命令行克隆

      git clone https://github.com/transitive-bullshit/nextjs-notion-starter-kit.git

    • 使用Github Desktop或者vscode克隆

  2. 修改 site.config.js

    两个选项

    // 获取你要分享的notion page的id (必须)
  rootNotionPageId: '78fc5a4b88d74b0e824e29407e9f1ec1',

// 生成网站的基础信息 (必须)
  name: 'Transitive Bullshit',//名称
  domain: 'transitivebullsh.it',//域名可不填
  author: 'Travis Fischer',//作者

  3. 安装node.js

  4. 安装yarn

    npm install -g yarn

  5. 修改npm 或yarn软件源

    npm config get registry  // 查看npm当前镜像源

npm config set registry https://registry.npm.taobao.org/  // 设置npm镜像源为淘宝镜像

yarn config get registry  // 查看yarn当前镜像源

yarn config set registry https://registry.npm.taobao.org/  // 设置yarn镜像源为淘宝镜像

# 部分**镜像源地址部分**
npm --- https://registry.npmjs.org/

cnpm --- https://r.cnpmjs.org/

taobao --- https://registry.npm.taobao.org/

nj --- https://registry.nodejitsu.com/

rednpm --- https://registry.mirror.cqupt.edu.cn/

npmMirror --- https://skimdb.npmjs.com/registry/

deunpm --- http://registry.enpmjs.org/

  6. yarn安装依赖包

    yarn install

  7. 本地测试

    yarn run dev

  8. 部署到 vercel

    npm run deploy

nobelium

  1. 复制这个 Notion 模板，开启share to web 选项
  2. Fork 或 clone 项目
  3. 修改 blog.config.js
  4. （可选）替换/public文件夹下的favicon.svg 、favicon.ico 
  5. 部署在 Vercel 上设置以下环境变量：
    • NOTION_PAGE_ID notion page id （必填）
    • NOTION_ACCESS_TOKEN（可选，不推荐）如果不开启share to web 选项，可以使用令牌让 Nobelium 从 Notion 数据库中抓取数据。令牌为 cookie 中为token_v2

PS： Notion 令牌的有效期只有 180 天，需要在 vercel 仪表板中手动更新，将来可能会切换到官方 API 来解决此问题。此外，Notion 数据库中的图像将无法正常显示。

部署到 vercel

  1. 进入官网
  2. 点击部署
  3. 选择使用github储存库进行部署（如果是clone的项目，请上传到自己的github仓库）
  4. 选择从 GitHub 中导入前面配置好的仓库，点击左侧的下拉框，选择「Add GitHub Org Account」。
  5. 从自己的 GitHub 中选择刚配置好的仓库，仓库的名称应为「nextjs-notion-starter-kit」，接着点击右侧蓝色的导入按钮「Import」。
  6. 按照页面提示，不需要修改配置信息，点击下一步，等待 Vercel 自动运行设定好的程序，直至页面返回「Successful」的提示，说明我们的博客顺利部署好了。
  7. 在返回的 Successful 页面，会有一个「Visit」的按钮，点击按钮就可以前往 Vercel 部署好的页面。
