使用 Stringbuffer 代替 字符串连接

String是在Java中最常用的类。特别是在 Web 应用程序中，它被广泛用于生成和格式化 HTML 内容。

String被设计为不可变的；为了修改一个 String，您必须创建一个新 String对象。因此，字符串连接可能会导致在构造 String最终对象之前创建许多中间对象 String。

StringBuffer是 的可变伴生类 String；它允许您修改 String. 因此，StringBuffer通常比 String需要连接时更有效。

如果在多个语句中重复附加到一个字符串，请使用 StringBuffer 而不是字符串连接

对 String重复使用“+=”操作代价高昂。

例如：

String s = new String();
     [do some work ...]
s += s1;
     [do some more work...]
s += s2;

用 StringBuffer 替换上面的字符串连接：

StringBuffer strbuf = new StringBuffer();
     [do some work ...]
strbuf.append(s1);
     [so some more work ...]
strbuf.append(s2);
String s = strbuf.toString();

如果连接在一个语句内，则使用 String 或 StringBuffer

String并 StringBuffer在某些情况下执行相同的操作；所以你不需要 StringBuffer直接使用。

	String s = "a" + "b" + "c";
to
	String s = "abc";

优化是由编译器自动完成的。

  • Java2编译器会自动折叠上面的。
  • Java2 编译器还会自动转换以下内容： 
            String s = s1 + s2;
    to
        String s = (newStringBuffer()).append(s1).append(s2).toString();
	# 在这些情况下，没有使用 `StringBuffer` 的必要。

如果知道字符串的大小，则使用 StringBuffer 而不是字符串连接

默认字符缓冲区为 StringBuffer16。当缓冲区已满时，必须重新分配一个新的字符缓冲区（通常是原始缓冲区大小的两倍）。将内容复制到新缓冲区后，旧缓冲区将被释放。如果创建的 StringBuffer 的缓冲区大小足以容纳 String.

以下将比使用 String 字符串连接更有效。

String s = (new StringBuffer(1024)).
append(s1).append(s2). toString();

将比

String s = s1 + s2;
