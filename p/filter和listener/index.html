Filter和Listener

FIlter 指过滤器

  1. 概念
  • 生活中的过滤器:空气过滤器 净水器
  • web中的过滤器:当访问服务器的资源时，过滤器可以将请求拦截下来，完成一些特殊的功能
  • 过滤器的一些功能: 一般用于完整通用的操作：
    • 比如登录验证 统一编码设置 敏感字符过滤

  1. 快速入门:

    • 步骤:

      1. 定义一个类 实现Filter接口，
      2. 复写方法
      3. 配置拦截资源:
      4. 注解配置 @WebFilter("/*")
      5. web.xml配置
    • @WebFilter("/*")
public class FilterDemo implements Filter {

	@Override
	public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
           throws IOException, ServletException {

       System.out.println("执行了filter");
       chain.doFilter(request, response);

	}

}

  2. 过滤器细节

    • web.xml

    • <filter>
    <filter-name>demo1</filter-name>
    <filter-class>com.edu.web.filter.FilterDemo</filter-class>
</filter>

<filter-mapping>
    <filter-name>demo1</filter-name>
    <!-- 拦截的请求路径 -->
    <url-pattern>/*</url-pattern>
</filter-mapping>

    • 过滤器执行流程

      • 执行 doFilter方法()
      • 对request对象进行 增强
      • chain.doFilter() 放行 执行对应的Servlet
      • 对response对象进行增强
      • 执行 doFilter后的方法

  3. 过滤器生命周期方法

    • init 应用服务器启动时执行一次
    • doFilter 每一次请求被拦截时 执行
    • destroy 在服务器关闭后，Filter对象被销毁。

  4. 过滤器配置详解

    • 拦截路径配置

      • 具体的资源路径 :1 /index.jsp 只有访问index.jsp时 过滤器才会被执行
      • 拦截目录 /user/* 访问user下的所有执行时过滤器被拦截
      • 拦截指定类型文件 .jsp 注意不加/
      • 拦截所有资源 /*
      • 拦截方式

    • 指的是资源被访问的方式:

      • 可以直接通过浏览器发送请求 资源
      • 可以在服务端通过 转发访问资源

  5. 注解配置:

    • 设置 WebFilter()

      • dispatcherType:

      • REQUEST 浏览器请求

      • FORWARD 转发

      • INCLUDE 包含访问资源

      • ERROR 错误跳转专员

      • ASYNC: 异步访问资源

        web.xml

        <filter-mapping>
                                <dispatcher>REQUEST(同上)</dispatcher>
                          </filter-mapping>

  6. 过滤器链

    • 一个项目里可以配置多个过滤器
      • 如果有两个过滤器 分别为过滤器A 过滤器B
      • 执行顺序 A 过滤器B 资源 过滤器B 过滤器A
    • 过滤器先后顺序：
      • 注解配置：按照类名的字符串比较规则，值小的先执行
      • Afilter和Bfilter 分别比较 A先执行
      • Web.xml里的配置 谁配置在上面 谁先执行。
